This is a project for internal use for scheduling recurring tasks.

Environment variables
---------------------

Various environment variables are required (wip).

The deploy (on Vercel, Netlify, etc.) needs:
- `NEXT_PUBLIC_CONVEX_URL`: the URL of the Convex deployment (of the form `https://xxx.convex.cloud`)
- `CONVEX_DEPLOY_KEY`: generated from the Convex dashboard
- `NEXT_PUBLIC_VAPID_PUBLIC_KEY`: public Vapid key
- `VAPID_PRIVATE_KEY`: private Vapid key, generated by installing `web-push` (via npm) and running `web-push generate-vapid-keys`

Uses of environment variables:
- `convex/auth.config.ts`: top level (unclear who runs this?)
- `convex/notifications.ts`: top level ("use node") and two actions
- `app/ConvexClientProvider.ts`: client (convex url)
- `components/usePushNotificationManager.ts`: (vapid public key)
