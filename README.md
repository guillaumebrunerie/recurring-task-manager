This is a project for internal use for scheduling recurring tasks.

Environment variables
---------------------

Various environment variables are required

The deploy (on Vercel, Netlify, etc.) needs:
- `CONVEX_DEPLOY_KEY`: generated from the Convex dashboard, used when deploying
- `NEXT_PUBLIC_CONVEX_CLOUD_URL`: the URL of the Convex deployment (of the form
  `https://xxx.convex.cloud`), sent to and used by the client to connect to the
  right Convex instance
- `NEXT_PUBLIC_VAPID_PUBLIC_KEY`: public Vapid key, sent to and used by the
  client to register subscriptions

The Convex deploy needs:
- `AUTH_GOOGLE_ID` and `AUTH_GOOGLE_SECRET`: OAuth with Google
- `JWKS` and `JWT_PRIVATE_KEY`: some auth stuff
- `SITE_URL`: the URL of the website, to redirect to after authenticating
- `CRON_NOTIFICATIONS_ENABLED`: whether to enable cron notifications (to disable
  them in the dev deploy)
- `VAPID_CONTACT_EMAIL`, `VAPID_PRIVATE_KEY`, and `VAPID_PUBLIC_KEY`: Vapid
  details

The private and public Vapid keys can be generated by installing `web-push` (via
npm) and running `web-push generate-vapid-keys`.
